# shodo-navi-final

iPhone Safariで使うことを最優先にした、現在地・目的地検索・徒歩ルート表示・自動再ルート対応の地図ナビPWAです。

PC上の `node server.js` や ngrok に依存せず、Vercel / Netlify / GitHub Pages などのHTTPS静的ホスティングへ公開して使う構成です。

正式な設計書は `docs/00_design.md` です。

## 1. アプリ概要

このアプリは、iPhone単独で次の流れを行うための地図アプリです。

1. Safariで公開URLを開く。
2. 現在地の利用を許可する。
3. 目的地を検索する。
4. 現在地から近い順の検索結果を見る。
5. 目的地を選ぶ。
6. 徒歩ルートを確認する。
7. ホーム画面に追加してアプリ風に使う。

地図表示にはLeaflet、地図タイルにはOpenStreetMap、目的地検索にはNominatim、徒歩ルートにはFOSSGIS OSRM foot routingを使います。

## 2. 主な機能

- Leafletによる地図表示
- iPhone Safariでの現在地取得
- 初回のみ現在地へ地図を移動
- 2回目以降の位置更新では地図を勝手に移動しない
- 現在地マーカーと精度円の表示
- 現在地修正ボタンによるGPS再取得
- 目的地検索
- 検索結果の現在地から近い順表示
- 検索結果ごとの距離表示
- 目的地マーカー表示
- 現在地から目的地までの徒歩ルート表示
- 徒歩距離と目安時間の表示
- ルートから外れた場合の自動再ルート
- 現在地修正後のルート更新
- PWA対応
- ホーム画面追加対応

## 4. iPhoneでの使い方

1. iPhoneのSafariで公開URLを開きます。
2. 位置情報の許可を求められたら「許可」を選びます。
3. 初回だけ現在地へ地図が移動します。
4. 地図は指で移動、ピンチで拡大縮小できます。
5. 下部の検索欄に目的地を入力して「検索」を押します。
6. 検索結果から目的地を選びます。
7. 目的地マーカーと徒歩ルートが表示されます。
8. 現在地がずれている場合は「現在地修正」を押します。
9. 目的地がある状態で現在地修正を押すと、新しい現在地からルートを更新します。
10. 表示中のルートから外れた場合は、自動で再ルートします。

## 5. ホーム画面への追加方法

1. iPhoneのSafariで公開URLを開きます。
2. 共有ボタンを押します。
3. 「ホーム画面に追加」を選びます。
4. 名前を確認して「追加」を押します。
5. ホーム画面のアイコンから起動します。

ホーム画面から起動すると、Safariの通常表示よりアプリに近い見た目で使えます。

## 6. 公開方法

位置情報、Service Worker、PWAはHTTPS環境で確認してください。ローカルファイルを直接開く方法は正式な確認方法にしません。

このアプリは静的ファイルだけで動きます。iPhone単独利用の本番手順では、PCで `node server.js` を起動し続ける必要はありません。ngrokのような一時URLにも依存しません。

### Vercel候補

1. GitHubなどにこのリポジトリを置きます。
2. Vercelで新規プロジェクトとして読み込みます。
3. Framework Presetは静的サイトとして扱います。
4. Build Commandは不要です。
5. Output Directoryはリポジトリ直下です。
6. 発行されたHTTPS URLをiPhone Safariで開きます。

### Netlify候補

1. Netlifyで新しいサイトを作成します。
2. このリポジトリを接続します。
3. Build Commandは空で構いません。
4. Publish Directoryはリポジトリ直下です。
5. 発行されたHTTPS URLをiPhone Safariで開きます。

### GitHub Pages候補

1. GitHubにこのリポジトリをpushします。
2. Repository SettingsからPagesを開きます。
3. Branchを選び、公開元をリポジトリ直下にします。
4. 発行されたHTTPS URLをiPhone Safariで開きます。

## 7. 検証項目

### 基本表示

- Safariで公開URLを開ける
- 地図が表示される
- 地図を指で移動できる
- ピンチで拡大縮小できる
- 下部パネルがSafe Areaに隠れない

### 現在地

- 位置情報許可ダイアログが出る
- 許可すると現在地マーカーが出る
- 初回だけ現在地へ地図が移動する
- 2回目以降の位置更新で地図が勝手に戻らない
- 現在地修正ボタンでGPSを再取得できる

### 検索

- 日本語で検索できる
- 検索結果が表示される
- 検索結果が現在地から近い順に並ぶ
- 距離がm/kmで表示される
- 検索失敗時にメッセージが出る

### ルート

- 検索結果をタップできる
- 目的地マーカーが表示される
- 徒歩ルート線が表示される
- 徒歩距離と目安時間が表示される
- ルート取得失敗時にメッセージが出る

### 再ルート

- ルートから約60m以上外れた場合に再ルートされる
- 再ルートが連発しない
- 目的地が保持される
- 再ルート失敗時に既存ルートが消えない
- 現在地修正後、目的地がある場合にルートが更新される

### PWA

- Safariの共有メニューからホーム画面に追加できる
- ホーム画面アイコンから起動できる
- 一度読み込んだ後、オフラインでも最低限アプリ画面が表示される
- 地図タイル、検索、ルートはオンライン時のみ動作する

## 8. セキュリティ注意点

- 位置情報はHTTPS環境で使う前提です。
- 位置情報はブラウザ上で地図表示、距離計算、ルート作成のために使います。
- 現在地履歴は保存しません。
- LocalStorageに位置情報を保存しません。
- 共有端末や他人の端末では、位置情報許可を残したままにしないよう注意してください。
- 公開URLを他人に共有しても、相手があなたの現在地を見られるわけではありません。位置情報は各端末のSafari許可に基づいて取得されます。
- APIキーは使いません。
- 検索結果の施設名や住所はHTMLとして挿入せず、文字列として表示します。
- エラー表示に内部情報やスタックトレースを出しません。
- 地図タイル、検索API、ルートAPIのレスポンスを長期保存しません。
- 外部サービスには利用制限があるため、高頻度利用や業務利用では専用APIへの切り替えを検討してください。
- Service Workerは静的ファイルをキャッシュします。更新時はキャッシュ名を変更して古いキャッシュを削除します。
